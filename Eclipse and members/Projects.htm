Projects

All work at Eclipse is done in sub-projects and components organized under top-level projects. The top-level projects are managed by PMCs (project management committees).

The Summarized Timeline lists all the planned and pending major releases. Each major release is preceded by a Release Review announced here and via RSS. The slides and IP logs of previous releases are available for inspection.

What are the top-level projects?

  • The Eclipse Top-Level Project - an open source, robust, full-featured, commercial-quality, industry platform for the development of highly integrated tools and rich client applications

  • The Eclipse Tools Project - a focal point for diverse tool projects on the Eclipse Platform in order to minimize overlap and duplication, ensure maximum sharing and creation of common components, and promote seamless interoperability between diverse types of tools.

  • The Eclipse Web Tools Platform Project - WTP provides a generic, extensible and standards-based platform and set of tools for J2EE and Web-centric application development. Download 1.0 now!

  • Test & Performance Tools Platform (TPTP) Project - TPTP provides powerful frameworks and services for an open platform upon which developers build unique test and performance tools—both open source and commercial—that easily integrate with Eclipse and other tools and address the entire test and performance lifecycle, from developer testing through production monitoring

  • The Business Intelligence and Reporting Tools (BIRT) Project - open source and extensible tools and frameworks for easily incorporating reporting functions within applications.

  • The Eclipse Data Tools Platform Project - provides extensible frameworks and exemplary tools, enabling a diverse set of plug-in offerings specific to particular data-centric technologies and supported by the DTP ecosystem.

  • Device Software Development Platform - DSDP provides an extensible, standards-based platform to support all phases of the device software (embedded) development process, including hardware bring-up, platform software development, and embedded application software development.

  • The Eclipse SOA Tools Platform Project - STP is an open source collaborative software development project dedicated to providing a generic, extensible, standards-based tool platform for producing Service Oriented Architecture (SOA) applications.

  • The Eclipse Technology Project (Incubators) - new channels for open source developers, researchers, academics and educators to participate in the on-going evolution of Eclipse. Organized in three streams: Research, Incubators and Education.

How do the projects operate?

The projects follow the Eclipse Development Process. Each PMC defines a top-level project-specific development process that interacts with the overall Eclipse Development Process.

At irregular times, there are inter-project committer gatherings.
